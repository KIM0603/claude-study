# STEP 3 — 본문 작성: 섹션별 단계적 생성 & 파일 저장

> **목표**: 7개 섹션을 하나씩 깊이 있게 작성하고 파일로 저장  
> **소요 시간**: 60~80분

---

## ⚠️ 가장 중요한 원칙

> **"기획서 전체 써줘" 는 틀린 방법입니다.**

한 번에 전체를 요청하면 각 섹션이 얕아집니다.  
**섹션 하나씩, 사람이 검토하면서** 진행하는 것이 핵심입니다.  
— 넥스트유니콘 AI 기획서 가이드 참고

---

## 3-1. 섹션별 작성 순서

아래 순서대로 하나씩 진행합니다. 각 섹션 작성 후 반드시 검토하세요.

### 섹션 1: 배경 및 문제 정의

```
"1. 배경 및 문제 정의 섹션만 먼저 써줘.
 시장 규모 데이터와 문제의 심각성을 수치로 포함해줘.
 섹션 끝에 '핵심 메시지 1줄'을 추가해줘."
```

**검토 포인트**
- [ ] 왜 지금 이 문제를 해결해야 하는가가 설명되었는가?
- [ ] 시장 규모 또는 문제 발생 빈도 수치가 있는가?
- [ ] 독자가 "맞아, 이건 진짜 문제야"라고 느낄 수 있는가?

---

### 섹션 2: 목적 및 KPI

```
"방금 쓴 배경 섹션을 참고해서, 
 2. 목적 및 KPI 섹션을 연결성 있게 써줘.
 KPI는 3개월 / 6개월 / 1년으로 나눠서 각각 3개씩.
 KPI는 반드시 측정 가능한 수치로."
```

**KPI 예시 형식**
```markdown
## 2. 목적 및 KPI

### 서비스 목적
[목적 1~2줄]

### 성공 지표

| 기간 | KPI | 목표치 | 측정 방법 |
|------|-----|--------|-----------|
| 3개월 | 월간 활성 사용자 | 500명 | GA4 |
| 3개월 | 커리큘럼 완료율 | 40% | 내부 DB |
| 6개월 | 유료 전환율 | 15% | 결제 시스템 |
| 1년 | 월간 반복 매출 | 1,000만원 | 회계 |
```

---

### 섹션 3: 타겟 유저 & 페르소나

```
"STEP2에서 만든 페르소나를 바탕으로
 3. 타겟 유저 & 페르소나 섹션을 완성해줘.
 유저 세그먼트도 함께 정리해줘."
```

---

### 섹션 4: 솔루션 및 핵심 기능

```
"4. 솔루션 및 핵심 기능 섹션을 써줘.
 우선순위별로 Must Have / Should Have / Nice to Have 
 3단계로 나눠서 정리해줘.
 각 기능에 '이 기능이 어떤 페인포인트를 해결하는지' 1줄씩."
```

---

### 섹션 5: 실행 계획 & 타임라인

```
"5. 실행 계획 & 타임라인 섹션을 써줘.
 Phase 1(1~3개월) / Phase 2(4~6개월) / Phase 3(7~12개월)로 구분.
 각 Phase의 주요 마일스톤과 담당 역할도 포함."
```

---

### 섹션 6: 리스크 & 대응 방안

```
"6. 리스크 & 대응 방안 섹션을 써줘.
 이 사업이 실패할 수 있는 이유 5가지를 먼저 제시하고,
 각각에 대한 구체적인 대응 방안을 작성해줘.
 발생 가능성(상/중/하)과 영향도(상/중/하)도 포함."
```

---

### 섹션 7: 기대 효과 & 결론

```
"7. 기대 효과 & 결론 섹션을 써줘.
 정량적 기대 효과 (수치 기반) 3가지,
 정성적 기대 효과 3가지,
 그리고 '지금 바로 시작해야 하는 이유' 1단락으로 마무리."
```

---

## 3-2. 컨텍스트 연결 요령

섹션을 연결할 때 이렇게 이전 내용을 참조하세요.

```
"방금 쓴 [섹션명]을 참고해서..."
"앞서 정의한 페르소나의 페인포인트와 연결해서..."
"KPI 달성을 위한 관점에서..."
```

---

## 3-3. 파일 저장

각 섹션이 완성될 때마다, 또는 전체 완성 후 파일로 저장합니다.

```
"지금까지 작성한 모든 섹션을 하나의
 마크다운 파일 'my-service-plan.md'로 저장해줘.
 상단에 목차도 자동으로 추가해줘."
```

Claude Code가 직접 파일을 생성해줍니다.
