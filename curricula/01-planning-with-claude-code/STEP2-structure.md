# STEP 2 — 구조 설계: Co-STAR로 배경·문제 정의하기

> **목표**: Co-STAR 프레임워크로 기획서의 핵심 뼈대(배경·페르소나·문제) 완성  
> **소요 시간**: 40~50분

---

## 2-1. Co-STAR 프레임워크로 배경 섹션 작성

### Co-STAR란?

브런치 기획 실무자들 사이에서 검증된 AI 기획서 프레임워크.  
"기획서 써줘"가 아닌 6가지 요소를 명확히 입력해서  
설득력 있는 결과를 만드는 방법입니다.

| 요소 | 의미 | 예시 |
|------|------|------|
| **C**ontext | 회사/서비스 배경 | "Claude 활용법을 가르치는 AI 교육 스타트업" |
| **O**bjective | 이 기획서의 목표 | "투자자·팀원 설득을 위한 서비스 기획서" |
| **S**tyle | 문서 스타일 | "2-Pager + 상세 기획서 2종 구성" |
| **T**one | 어조 | "데이터 기반, 설득적, 전문적" |
| **A**udience | 독자 | "시드 투자자, 공동창업자 후보" |
| **R**esponse | 출력 형식 | "마크다운 섹션별, 각 섹션 끝에 근거 데이터 포함" |

---

### Co-STAR 프롬프트 템플릿

```
다음 조건으로 기획서의 배경 및 문제 정의 섹션을 작성해줘.

Context: [회사/서비스 배경 — 무엇을 하는 서비스인가]
Objective: [이 기획서로 달성할 목표 — 왜 쓰는가]
Style: [문서 스타일 — 2-Pager / 상세 기획서 / 제안서]
Tone: [어조 — 공식적 / 설득적 / 데이터 중심]
Audience: [독자 — CEO / 투자자 / 팀원 / 팀장]
Response: [출력 형식 — 마크다운 / 항목별 / 표 포함]
```

---

### 실전 예시 — AI 교육 플랫폼 기준

```
다음 조건으로 기획서의 배경 및 문제 정의 섹션을 작성해줘.

Context: Claude AI 활용법을 실전 커리큘럼으로 가르치는 
         에이전트 기반 AI 교육 스타트업. 
         에이전트가 리서치부터 실습까지 단계별로 안내함.

Objective: 시드 투자자와 공동창업자를 설득하기 위한 
           서비스 런칭 기획서 작성

Style: 첫 페이지는 2-Pager 요약본, 이후 섹션별 상세 기획서

Tone: 데이터와 시장 수치 기반, 설득적, 전문적

Audience: AI/EdTech에 관심 있는 시드 투자자, 
          업무 자동화에 관심 있는 직장인

Response: 마크다운 형식, 각 주장에 시장 데이터나 
          근거 수치 포함, 섹션 끝에 핵심 메시지 1줄
```

## 2-2. 타겟 페르소나 & 문제 정의 섹션 작성

### 페르소나 프롬프트 (스파르타 AI 블로그 검증 방법)

AI로 페르소나를 만들 때 이 7가지를 반드시 포함하면  
"실제 사람처럼 느껴지는" 페르소나가 나옵니다.

```
내 서비스의 핵심 유저 페르소나를 2개 만들어줘.

각 페르소나에 다음 7가지를 포함해:
1. 인구통계: 나이, 직업, 현재 상황
2. JTBD: 이 서비스를 쓰는 이유 (진짜 목적)
3. 트리거: 처음 사용을 시작하게 되는 구체적 계기
4. 장벽: 사용을 주저하게 되는 이유
5. 현재 대안: 지금은 어떻게 해결하고 있는가
6. 한 줄 인용문: 실제 말투로 (예: "매번 기획서 쓸 때마다...")
7. 핵심 페인포인트 3가지: 구체적 상황으로
```

---

### 문제 정의 검증 프롬프트

페르소나를 만든 후, 이 프롬프트로 문제의 크기를 수치화합니다.

```
위 페르소나가 겪는 문제를 다음 관점에서 정량화해줘:

1. 이 문제가 얼마나 자주 발생하는가? (빈도)
2. 한 번 발생 시 소요되는 시간/비용은? (크기)
3. 이 문제를 겪는 사람이 국내에 몇 명이나 되는가? (시장 규모)
4. 현재 대안 솔루션의 한계는 무엇인가?
5. 우리가 이 문제를 해결하기에 적합한 이유는?
```

---

### ✅ 체크리스트

- [ ] Co-STAR 6요소를 내 서비스에 맞게 채웠다
- [ ] 배경 섹션에 시장 데이터가 포함되었다
- [ ] 페르소나 2개가 완성되었다
- [ ] 각 페르소나의 JTBD가 명확하게 정의되었다
- [ ] 문제의 크기가 수치로 표현되었다
