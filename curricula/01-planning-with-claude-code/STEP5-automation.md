# STEP 5 — 스킬화: 두 번째 기획서는 30분 안에 나온다

> **목표**: 이번에 배운 모든 방법을 Claude Code 스킬 파일로 저장  
> **소요 시간**: 25~30분

---

## 스킬이란?

Claude Code의 `.claude/skills/` 폴더에 `SKILL.md`를 만들면  
다음 프로젝트에서 **"기획서 작성 스킬 써줘"** 한 마디만 해도  
이번 커리큘럼에서 배운 모든 방법이 자동으로 적용됩니다.

> **"MCP가 주방이라면, Skills는 레시피다"**  
> — highoutputclub 블로그

---

## 5-1. 스킬 폴더 구조 만들기

```bash
# Claude Code 안에서 실행
"다음 폴더 구조를 만들어줘:

.claude/skills/planning/
├── SKILL.md          ← 메인 지침 파일
├── template.md       ← 기획서 빈 템플릿
├── cpo-prompts.md    ← CPO 검증 프롬프트 모음
└── examples/         ← 이번에 만든 기획서 예시
    └── sample-plan.md"
```

---

## 5-2. SKILL.md 내용 (복사해서 사용)

```markdown
# 기획서 작성 스킬

## 스킬 설명
Claude Code에서 탄탄한 기획서를 작성하는 표준 워크플로우.
Co-STAR 프레임워크 + Plan Mode + CPO 검증을 자동으로 적용.

## 트리거 키워드
다음 키워드 감지 시 이 스킬을 자동 적용:
- "기획서", "계획서", "제안서", "planning", "PRD"

## 자동 실행 순서

### Phase 1: 준비 (5분)
1. "어떤 서비스/프로젝트의 기획서인지 알려줘"
2. CLAUDE.md의 프로젝트 컨텍스트 확인
3. Plan Mode(Shift+Tab)로 목차 초안 제안 + 승인

### Phase 2: 구조 설계 (20분)
4. Co-STAR 6요소 확인 (없으면 질문)
5. 타겟 페르소나 2개 생성
6. 문제 크기 수치화

### Phase 3: 본문 작성 (60분)
7. 섹션 1부터 7까지 하나씩 작성
8. 각 섹션 완료 후 저장 요청
9. 섹션 간 컨텍스트 연결 유지

### Phase 4: 검증 (30분)
10. CPO 역할 날카로운 질문 10개
11. 실패 케이스 5가지 검증
12. 논리 흐름 4가지 평가

### Phase 5: 최종 저장 (5분)
13. 전체 기획서를 단일 마크다운 파일로 저장
14. 목차 자동 생성
15. "완성됐어. 파일명: [서비스명]-plan-[날짜].md"

## 파일 저장 규칙
- 파일명: {서비스명}-plan-{YYYY-MM-DD}.md
- 저장 위치: 현재 작업 폴더
- 형식: 마크다운 (목차 포함)

## 작성 규칙
- 섹션 하나씩 진행 (한 번에 전체 금지)
- 각 섹션 완료 후 사람이 검토
- 수치 없는 주장 작성 금지
- 애매한 표현("검토 필요" 등) 금지
```

---

## 5-3. 스킬 사용 방법

스킬이 설치된 이후에는 이렇게만 하면 됩니다:

```
# 새 프로젝트에서
claude
> "기획서 써줘. 서비스는 [서비스명]이야."

← Claude가 자동으로 스킬을 감지하고
  Phase 1부터 단계별로 진행합니다
```

---

## 5-4. 스킬 업그레이드 방법

기획서를 작성하면서 유용한 프롬프트를 발견하면 즉시 저장하세요:

```
"방금 쓴 CPO 검증 프롬프트가 효과적이었어.
 cpo-prompts.md 파일에 추가해줘."
```
