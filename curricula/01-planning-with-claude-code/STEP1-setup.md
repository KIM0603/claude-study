# STEP 1 — 기반 세팅: CLAUDE.md에 기획자 DNA 심기

> **목표**: Claude Code가 매 세션마다 기획자의 규칙과 스타일을 기억하도록 설정  
> **소요 시간**: 30~40분

---

## 1-1. 프로젝트 폴더 + CLAUDE.md 생성

### 왜 이 단계가 중요한가?

CLAUDE.md 없이 작업하면 매 대화마다 컨텍스트를 다시 설명해야 합니다.  
기획서 스타일, 회사/프로젝트 맥락, 금지 사항을 한 번만 설정하면  
Claude가 항상 그 기준을 지키며 작업합니다.

### 실행

```bash
# 터미널에서
mkdir my-planning && cd my-planning
claude  # Claude Code 실행
```

```
# Claude Code 안에서
/init   # CLAUDE.md 자동 생성
```

---

### CLAUDE.md 템플릿 (복사해서 사용)

```markdown
# 기획서 작성 규칙

## 역할
나는 시니어 기획자 역할로 행동한다.
항상 "왜 해야 하는가"를 먼저 쓴다.
데이터와 근거 없는 주장은 하지 않는다.

## 프로젝트 컨텍스트
- 서비스명: [여기에 입력]
- 타겟 유저: [여기에 입력]
- 핵심 문제: [여기에 입력]
- 독자: [투자자 / 팀원 / CEO 등]

## 기획서 필수 구조
1. 배경 및 문제 정의 (Why)
2. 목적 및 목표 (KPI 포함)
3. 타겟 유저 & 페르소나
4. 솔루션 및 핵심 기능
5. 실행 계획 & 타임라인
6. 리스크 & 대응 방안
7. 기대 효과 & 성공 지표

## 작성 규칙
- 각 섹션은 독립적으로 완결성을 가져야 한다
- KPI는 반드시 측정 가능한 수치로 작성한다
- 타임라인은 3개월 / 6개월 / 1년으로 구분한다
- 경쟁사 분석은 최소 3개 이상 포함한다

## 금지 사항
- 근거 없는 수치 사용 금지
- 결론 없는 나열 금지
- "검토 필요", "추후 논의" 같은 애매한 표현 금지
- 한 번에 전체 기획서를 쓰지 않는다 (섹션별로 진행)

## 파일 저장
- 작성 완료된 섹션은 planning.md에 누적 저장
- 섹션 완료 시 "지금까지 내용 저장해줘"라고 요청
```

### 커스터마이징 포인트

- `프로젝트 컨텍스트` 섹션을 내 프로젝트에 맞게 반드시 수정
- `금지 사항`은 본인 스타일에 맞게 추가/제거 가능
- 회사 특유의 기획서 양식이 있으면 `기획서 필수 구조`에 반영

## 1-2. Plan Mode로 기획 구조 먼저 검토받기

### Plan Mode란?

Claude Code에서 **Shift+Tab**으로 활성화.  
코드(또는 문서)를 바로 작성하지 않고 먼저 계획을 세워서 보여줍니다.  
기획서 작성에서는 **"어떤 구조로 쓸지"를 먼저 승인받는** 용도로 사용합니다.

### 실행 방법

```
# Claude Code 안에서
Shift+Tab    ← Plan Mode 활성화 (화면에 표시됨)

그 다음 입력:
"AI 교육 플랫폼 기획서를 작성할 예정이야.
 어떤 구조로 쓰면 가장 설득력 있을지
 목차 초안을 먼저 제안해줘."
```

### Plan Mode 활용 팁

```
# 구조 제안을 받은 후 검증 질문
"CPO 관점에서 이 목차에 빠진 게 있어?"

"투자자가 이 기획서를 보면 가장 먼저
 물어볼 것 같은 질문이 뭐야?"

"이 목차 순서가 스토리텔링 측면에서 
 가장 설득력 있는 순서야?"
```

- [ ] Plan Mode로 기획서 목차 초안을 받았다
