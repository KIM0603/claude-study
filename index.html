커리큘럼›01. Claude Code로 탄탄한 기획서 완성하기
단계별 실습 가이드
✅ 완성5단계 · 3~4시간비개발자 가능
CLAUDE.md에 기획자 DNA 심기
Claude Code가 매 세션마다 기획자의 규칙을 기억하도록 설정합니다
⏱ 30~40분
왜 이 단계가 중요한가? CLAUDE.md 없이 작업하면 매 대화마다 컨텍스트를 다시 설명해야 합니다. 한 번 설정하면 Claude가 항상 그 기준을 지킵니다.
1-1. 프로젝트 폴더 + /init
# 터미널에서 mkdir my-planning && cd my-planning claude # Claude Code 실행 # Claude Code 안에서: /init # CLAUDE.md 자동 생성
1-2. CLAUDE.md 핵심 내용
# 기획서 작성 규칙 ## 역할 나는 시니어 기획자 역할로 행동한다. 항상 "왜 해야 하는가"를 먼저 쓴다. ## 기획서 필수 구조 1. 배경 및 문제 정의 2. 목적 및 목표 (KPI 포함) 3. 타겟 유저 & 페르소나 4. 솔루션 및 핵심 기능 5. 실행 계획 & 타임라인 6. 리스크 & 대응 방안 7. 기대 효과 & 결론 ## 금지 사항 - 근거 없는 수치 사용 금지 - 한 번에 전체 기획서 작성 금지
1-3. Plan Mode로 목차 구조 검토
Shift+Tab ← Plan Mode 활성화 "기획서를 작성할 예정이야. 어떤 구조로 쓰면 가장 설득력 있을지 목차 초안을 먼저 제안해줘." # 받은 후 추가 검증: "CPO 관점에서 이 목차에 빠진 게 있어?"
✅ 체크리스트
✓
CLAUDE.md 파일이 생성되었다
✓
프로젝트 컨텍스트를 내 상황에 맞게 수정했다
✓
Plan Mode로 기획서 목차 초안을 받았다
✓
CPO 관점 검토까지 완료했다
Co-STAR 프레임워크로 배경·문제 정의
6가지 요소를 명확히 입력해서 설득력 있는 뼈대를 만듭니다
⏱ 40~50분
Co-STAR란? "기획서 써줘" 대신 Context·Objective·Style·Tone·Audience·Response 6가지를 명확히 입력합니다. 브런치 기획 실무자들 사이에서 검증된 프레임워크입니다.
2-1. Co-STAR 배경 섹션 프롬프트
다음 조건으로 배경 및 문제 정의 섹션을 작성해줘. Context: [회사/서비스 배경] Objective: [이 기획서로 달성할 목표] Style: [2-Pager / 상세 기획서] Tone: [설득적 / 데이터 중심] Audience: [투자자 / 팀원 / CEO] Response: [마크다운, 각 주장에 근거 데이터 포함]
2-2. 페르소나 생성 프롬프트
핵심 유저 페르소나 2개를 만들어줘. 각 페르소나에: - JTBD: 진짜 사용 이유 - 트리거: 사용 시작 계기 - 장벽: 주저하는 이유 - 한 줄 인용문 (실제 말투) - 핵심 페인포인트 3가지
✅ 체크리스트
✓
Co-STAR 6요소를 내 서비스에 맞게 채웠다
✓
배경 섹션에 시장 데이터가 포함되었다
✓
페르소나 2개가 완성되었다
✓
각 페르소나의 JTBD가 명확히 정의되었다
섹션별 단계적 본문 작성
⚠️ "전체 써줘"는 틀린 방법 — 섹션 하나씩, 검토하면서 진행
⏱ 60~80분
가장 많이 하는 실수: 한 번에 전체를 요청하면 각 항목이 얕아집니다. 섹션 하나씩 집중해서 작성하고, 사람이 검토한 뒤 다음으로 넘어가는 것이 핵심입니다.
3-1. 섹션별 순서 작성
"1. 배경 및 문제 정의 섹션만 먼저 써줘. 시장 데이터와 수치로 포함해줘." # 검토 후: "방금 쓴 배경 섹션을 참고해서, 2. 목적 및 KPI 섹션을 연결성 있게 써줘. KPI는 3개월/6개월/1년으로." # 7개 섹션을 이렇게 하나씩 진행
3-2. 파일 저장
"지금까지 모든 섹션을 my-service-plan.md 로 저장해줘. 상단에 목차도 자동으로 추가해줘."
✅ 체크리스트
✓
7개 섹션이 모두 작성되었다
✓
각 섹션을 독립적으로 검토했다
✓
KPI가 모두 측정 가능한 수치다
✓
파일이 저장되었다 (my-service-plan.md)
CPO·투자자 역할로 기획서 구멍 찾기
기획서의 치명적 약점은 본인이 못 찾습니다
⏱ 40~50분
핵심 원칙: CPO가 날카롭게 질문한 것 중 답변이 약한 부분 = 기획서에서 보강해야 할 섹션입니다.
4-1. CPO 역할 검증
당신은 시리즈B 스타트업의 CPO입니다. 기획서를 읽고 전사 리뷰에서 실제로 할 법한 날카로운 질문 10개 + PO 입장에서 각 답변을 작성해주세요. [기획서 내용 붙여넣기]
4-2. 실패 케이스 검증
이 기획이 실패할 수 있는 이유 10가지. 각 이유에 발생 가능성(상/중/하) + 보완 방법.
4-3. 논리 흐름 최종 평가
4가지를 5점 만점으로 평가: 1. 문장 구조 / 2. 논리 흐름 3. 가독성 / 4. 설득력 항목별 개선점을 구체적으로.
✅ 체크리스트
✓
CPO 검증 완료 (질문 10개 + 답변)
✓
약했던 섹션을 보강했다
✓
실패 케이스 확인 후 리스크 섹션 강화
✓
논리 흐름 평가에서 4점 이상을 받았다
스킬화 — 두 번째 기획서는 30분 안에 나온다
이번에 배운 모든 방법을 Claude Code 스킬 파일로 저장합니다
⏱ 25~30분
스킬이란? .claude/skills/ 폴더에 SKILL.md를 만들면 다음 프로젝트에서 "기획서 써줘"만 해도 5단계가 자동으로 적용됩니다.
5-1. 스킬 폴더 구조
.claude/skills/planning/ ├── SKILL.md # 메인 지침 (5단계 자동화) ├── template.md # 기획서 빈 템플릿 ├── cpo-prompts.md # 검증 프롬프트 6종 └── examples/ # 이번에 만든 기획서
5-2. 다음 기획서 사용법
"기획서 써줘. 서비스는 [서비스명]이야." ← Claude가 자동으로 스킬 감지 → 5단계를 순차적으로 진행합니다 🎉
